```
POST /api/user/api-keys
```

Creates a new API key scoped to your user account (and its organizations). To create a key owned by an organization instead, see [Create Organization API Key](/docs/api/api-keys/create-org).

<Callout type="warn">This endpoint accepts your logged-in dashboard **session** or an **unrestricted (full-access) user API key**. Scoped keys and organization keys are rejected — so a scoped key cannot mint new keys, but a session or full-access key can. Send your session cookie or `Authorization: Bearer` header with the request.</Callout>

### Request Body

<TypeTable
  type={{
    name: {
      description: 'A label for the key',
      type: 'string',
      required: true,
    },
    expiresIn: {
      description: 'Seconds until the key expires. Omit for a key that never expires.',
      type: 'number',
      required: false,
    },
  }}
/>

### Response

Returns the created API key object.

<TypeTable
  type={{
    id: {
      description: "The key's identifier",
      type: 'string',
    },
    key: {
      description: 'The secret key value',
      type: 'string',
    },
    name: {
      description: 'The label for the key',
      type: 'string',
    },

      description: 'A non-secret prefix of the key for display',
      type: 'string',
    },
    prefix: {
      description: 'The key prefix',
      type: 'string',
    },
    userId: {
      description: 'The ID of the user that owns the key',
      type: 'string',
    },
    enabled: {
      description: 'Whether the key is currently enabled',
      type: 'boolean',
    },
    expiresAt: {
      description: 'Expiration timestamp, or null if the key never expires',
      type: 'string | null',
    },
    createdAt: {
      description: 'Creation timestamp',
      type: 'string',
    },
    updatedAt: {
      description: 'Last update timestamp',
      type: 'string',
    },
    rateLimitEnabled: {
      description: 'Whether rate limiting is enabled for the key',
      type: 'boolean',
    },
    rateLimitMax: {
      description: 'Requests allowed per window',
      type: 'number',
    },
    rateLimitTimeWindow: {
      description: 'Window length in milliseconds',
      type: 'number',
    },
  }}
/>

**Note:** Additional fields may be present in the response object.
