```
GET /api/sites/:site/page-titles
```

Returns page-view metrics aggregated by HTML page title.

### Path Parameters

<TypeTable
  type={{
    site: {
      description: 'Site ID or site identifier',
      type: 'string | number',
      required: true,
    },
  }}
/>

### Query Parameters

Accepts all [Common Parameters](/docs/api/getting-started#common-parameters) (time and filters).

<TypeTable
  type={{
    limit: {
      description: 'Maximum number of page titles to return',
      type: 'number',
      default: 10,
    },
    page: {
      description: 'Page number (1-indexed). When provided, the response is paginated and includes a total count',
      type: 'number',
      required: false,
    },
  }}
/>

### Response

<TypeTable
  type={{
    value: {
      description: 'The page title',
      type: 'string',
    },
    pathname: {
      description: 'A representative path for the title',
      type: 'string',
    },
    count: {
      description: 'Unique sessions that viewed the title',
      type: 'number',
    },
    percentage: {
      description: 'Share of total',
      type: 'number',
      typeDescription: 'Range: 0-100',
    },
    pageviews: {
      description: 'Total pageviews',
      type: 'number',
    },
    bounce_rate: {
      description: 'Bounce rate percentage',
      type: 'number',
      typeDescription: 'Range: 0-100',
    },
    time_on_page_seconds: {
      description: 'Average time on page in seconds',
      type: 'number',
    },
  }}
/>

When the `page` parameter is supplied, the response is paginated: `data` becomes an object `{ data: [...], totalCount: number }`.
